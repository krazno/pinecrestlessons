# School isolation (required)

This repository hosts **separate, unrelated school projects**. They share hosting only — not brand, copy, assets, colors, or lesson content.

## Rules

1. **One school per task.** When you edit or build, work inside that school’s folder only unless the user explicitly asks for repo-wide hosting changes (e.g. `vercel.json`, root `index.html`).
2. **Never cross-reference schools.** Do not mention, link to, or reuse assets from another school’s folder in copy, comments, or UI.
3. **If the target school is unclear, ask.** Example: “Should this change go in `ursuline/` or `pinecrest-student/`?”
4. **Default context** is the school the user is currently working on (path or message).

## School folders (do not mix)

| Folder | School | Route |
|--------|--------|-------|
| `pinecrest-student/`, `pinecrest-teacher/`, `pinecrest/` | Pine Crest (Florida) | `/pinecrest-student/`, `/pinecrest-teacher/` |
| `app/ursuline-ai/`, `public/artifacts/` | Ursuline Academy Dedham (Massachusetts) | `/ursuline-ai/` |

Each folder has its own `SCOPE.md` with school-specific notes. Read `SCOPE.md` in the folder you are editing before making changes.

## Root homepage

`index.html` at the repo root is intentionally **blank** (single neutral color). It is not a school landing page. School experiences live only under their paths (e.g. `/ursuline-ai/`).
